## 基本规范
- 你所有的回复、代码注释和文档均应使用**简体中文**
- 对于关键逻辑、复杂算法和业务规则，必须添加注释，说明“为什么选择这么做”
- 开发的各功能应高度模块化，保障其未来的可维护性


## 开发语言
- 前端框架：React + Next.js (App Router)
- 样式方案：Tailwind CSS + Shadcn UI
- 状态管理：Zustand + React Hooks
- 后端/BaaS：Supabase


## 核心文件

### \documents\PRD.md 为说明文档，你应该始终遵循其中的需求
- 当检查到现有代码与需求冲突时，你应当向用户询问现在应该“修改需求”还是“修改代码”
- 若需修改需求，你应当在取得用户许可后才能对该文件的内容进行修改

### \documents\layout.md 为程序根目录下文件结构的说明
- 你应当在新建或重构了文件目录后，维护该文件的内容
