# hhwx Claude 规则

本目录用于统一管理 hhwx 的 Claude Code 项目规则。

## 使用原则

- 这里只保留长期有效、可复用、可验证的项目规则。
- 通用规则放在 .claude/CLAUDE.md 与无 paths 的规则文件中。
- 仅对特定目录或文件类型生效的要求，放入 .claude/rules/ 并使用 paths 约束，避免把无关规则注入所有任务。
- 若新增规则与现有规则冲突，应先收敛旧规则而不是继续叠加。

## 全局要求

- 与项目相关的回复、代码注释和文档默认使用简体中文；只有在用户或外部接口明确要求时才切换语言。
- 新功能和重构优先保证模块边界清晰，避免把多个职责耦合进同一个组件、Hook、路由或服务模块。
- 修改现有文件时，不做与当前任务无关的大规模格式化、重命名或结构迁移。新增代码应始终遵循命名规范；当前任务所触及的代码如果命名明显违反规范，可以顺手修正。
- 新增或重构目录结构后，必须同步更新 documents/layout.md。

## 规则组织

- 规则按主题拆分到 .claude/rules/ 目录下，一个文件对应一个主题。
- 没有 paths frontmatter 的规则全局生效；有 paths 的规则仅在处理匹配文件时加载。
- 具体规则清单以 .claude/rules/ 目录内容为准，此处不再逐一罗列文件名。

## 维护约束

- 规则应优先描述“默认做法”和“明确例外”，避免只写模糊口号。
- 如果某条规则只服务于一次性任务，不要写入 .claude。
- 如果某条规则已经失效，应直接更新或删除，不保留过时约定。
